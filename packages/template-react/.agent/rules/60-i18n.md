# i18n

The template ships with **i18next** + **react-i18next** preconfigured for English (`en`, default) and Simplified Chinese (`zh`). Initialization happens in [src/shared/i18n/index.ts](../../src/shared/i18n/index.ts) and is loaded once from [src/main.tsx](../../src/main.tsx).

## Hard rule

- **No user-visible string is hard-coded in JSX or in `aria-*` labels.** Every visible string goes through `useTranslation()`.
- Internal error messages, log messages, and developer-only debug strings (e.g. `throw new Error(...)`) MAY be in English plain text — they are not user-visible.

## Key naming

- Translation keys are dotted, namespaced by feature: `<feature>.<element>`.
  - `home.title`, `home.cta`
  - `counter.title`, `counter.toastIncreased`
- Avoid deep nesting. Two levels is usually enough; three is the practical maximum.
- Use **camelCase** for the leaf segment (`toastIncreased`, not `toast_increased`).
- Interpolated values use double-brace placeholders inside the value, **not** in the key: `"Current value: {{value}}"`.

## File layout

```
src/shared/i18n/
├── index.ts                        # i18next init + exports
└── locales/
    ├── en.json
    └── zh.json
```

Both locale files MUST contain the exact same set of keys. When you add a key in one, add it in all. (CI may enforce this later via `i18next-parser`.)

## Adding a new key

1. Decide which feature namespace it belongs to. If unsure, default to `<currentFeature>.<element>`.
2. Add the key to **every** locale file under `src/shared/i18n/locales/`.
3. Use it in the component: `const { t } = useTranslation(); ... {t('feature.key')}`.
4. If the key is dynamic (`{{value}}`), call `t('feature.key', { value })`.

See [skills/add-i18n-keys/SKILL.md](../skills/add-i18n-keys/SKILL.md) for the full playbook.

## Plurals and gender

i18next handles plurals via suffixes (`_one`, `_other`). Only introduce them when actually needed; do not preemptively suffix every key.

## Do not

- Do not introduce a new i18n library. Stay on i18next.
- Do not place translation strings inside component files as constants — they must live in `locales/*.json`.
- Do not concatenate translated strings (`t('a') + ' ' + t('b')`). Compose meaning at the i18n layer with a single key + interpolation.
