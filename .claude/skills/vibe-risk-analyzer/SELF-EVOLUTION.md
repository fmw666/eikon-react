# Self-Evolution Protocol（自进化协议）— v1

**目的**：让 skill 从每次真实运行中沉淀改进,但**绝不静默自改**——
进化只有一条路：候选 → 提醒用户 → 用户同意发提案 → 过 [CHANGE-GATE.md](CHANGE-GATE.md) → 落地。

**触发**：每次扫描（Phase 1–6）结束时,以及由本 skill 驱动的修复（remediation）收尾时,
执行下方反思清单（≤2 分钟,无沉淀则零输出,不打扰用户）。

---

## 反思清单（逐条过,只认"本次运行中实际发生"）

| # | 问题 | 沉淀信号示例 |
|---|---|---|
| E1 | 有无子项**误判**（误报/漏报/被博弈),且根因在规则而非项目特殊性? | 注释完备的静默 catch 被判为坏味道;拆文件躲阈值导致带数上升 |
| E2 | 有无工具/步骤**断点**是现有 fallback 链没接住、靠临场发明救场的? | 某环境下工具空转;手写了一个本应内置的脚本 |
| E3 | 有无**真实风险**任何现有维度都装不下? | 一类缺陷反复出现但无处扣分 |
| E4 | 有无对某**阈值**的校准证据（明显过严/过宽）? | 同类项目连续触发同一边界争议 |
| E5 | 本次为绕过 skill 缺陷做的**临时口径**,下次还会需要吗? | scope_note 里又写了一段"特殊解释" |

**不算沉淀**（就地分流,不进候选）：单项目特例（→ 该项目 `.vibe-riskrc.json` / 台账 / AGENTS.md）、
没有当场证据的设想、措辞偏好。

## 候选登记区（先进先出,上限 10 条;超限挤掉最旧未复现项）

> 格式：`- [日期] E# 一句话(项目匿名化) | 证据指认 | 复现计数`
> **复现 ≥2（不同项目）= 最强提案信号**;单项目证据只能留在这里观察,不得直接改规范文（Gate 2）。

## 运行结束时的用户提醒（有沉淀才出现）

在最终报告/总结的末尾追加一小节,格式固定：

```
🧬 进化候选（N 条,来自本次运行）
1. [E2] <一句话> — 证据：<可指认的命令/输出>
...
其中 <K> 条与候选区既有条目复现匹配（已 ×2,达到提案门槛）。
要发起一份 PROPOSAL 吗?（将按 CHANGE-GATE 五关自查后落地）
```

用户未回应/拒绝 → 候选保留在登记区等待复现,**不得**自行修改任何规范文件。

## 硬性边界

1. 扫描/修复运行中**禁止**编辑 skill 的任何规范文件（SKILL.md / scoring-reference /
   模板 / scripts）;唯一例外是经用户当场明示同意的提案落地,且仍须过全部门控。
2. 本文件自身的修改同样走提案 + 门控。
3. 候选区是本文件唯一允许在运行中追加的区域,且受 10 条上限约束（Gate 3）。
